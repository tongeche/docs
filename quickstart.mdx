## Use this documentation by role

### Product and operations readers

1. Start on [TIBS documentation](/).
2. Open [Product overview](/product/overview) to find the relevant domain.
3. Move to [API introduction](/api-reference/introduction) if you need
   endpoint-level guidance.
4. Check [Legal center](/legal/index) for policy, privacy, and contractual
   documents.

### API integrators

1. Read [API introduction](/api-reference/introduction).
2. Use [API domain map](/api-reference/domain-map) to find the correct
   endpoint family.
3. Cross-check the matching product page to understand intent and boundaries.
4. Treat the domain map as the current source of truth until the full OpenAPI
   spec is published.

### Legal and compliance reviewers

1. Start in [Legal center](/legal/index).
2. Review the draft document structure for privacy, terms, and data
   processing.
3. Replace draft placeholders with approved legal text before public
   publication.

### Documentation contributors

1. Install the Mintlify CLI with `npm i -g mint`.
2. From the docs repo root, run `mint dev`.
3. Open `http://localhost:3000` to preview the site.
4. Run `mint broken-links` before pushing changes.

## Current scope

This site is being repositioned to serve three documentation jobs:

- Product documentation for the TIBS platform
- API and data-surface documentation derived from source repos
- Legal documentation for policies, terms, and regulated disclosures
