<Info>
  **Prerequisites**:
  - Node.js version 19 or higher
  - A docs repository with a `docs.json` file
  - Access to the source repositories you want documented
</Info>

Use this page when you are maintaining the TIBS docs site itself.

<Steps>
  <Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```

  </Step>

  <Step title="Preview locally">

Navigate to the repo root where `docs.json` is located, then run:

```bash
mint dev
```

A local preview of the documentation will be available at
`http://localhost:3000`.

  </Step>
</Steps>

## Documentation structure

- `index.mdx`: landing page
- `quickstart.mdx`: role-based orientation
- `product/`: product and workflow documentation
- `legal/`: privacy, terms, and compliance pages
- `api-reference/`: API overview and endpoint mapping
- `references/`: raw working material and internal source notes

## Working rules

- Derive technical facts from the selected product repositories.
- Do not leave Mintlify starter copy in published pages.
- Do not invent legal language, compliance claims, or retention policies.
- Mark draft legal pages clearly until approved text is available.
- Prefer one source of truth per topic, then link to it from adjacent pages.

## Validation

Validate link integrity before pushing:

```bash
mint broken-links
```

## Repository-driven documentation

This docs site is intended to document multiple TIBS repositories from a single
Mintlify instance. The expected flow is:

1. Confirm the source repositories.
2. Read each repo for setup, architecture, interfaces, and operational rules.
3. Publish stable docs here instead of scattering them across READMEs.
4. Keep repo-specific implementation details linked back to the correct source.

## Custom ports

By default, Mintlify uses port 3000. You can choose another port:

```bash
mint dev --port 3333
```

If port 3000 is already busy, Mintlify will try the next available port.

## Updating the CLI

If your preview behaves differently from production, update the CLI:

```bash
npm update -g mint
```

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module'>

This may be due to an outdated version of Node. Try the following:

1. Remove the currently installed version of the CLI: `npm remove -g mint`
2. Upgrade to Node v19 or higher.
3. Reinstall the CLI: `npm i -g mint`

  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">

Solution: Delete the `~/.mintlify` folder, then run `mint dev` again.

  </Accordion>
</AccordionGroup>
