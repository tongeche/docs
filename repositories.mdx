## What belongs here

This docs site is meant to consolidate stable documentation that would
otherwise be scattered across multiple code repositories.

Typical repo-derived content includes:

- Setup and environment requirements
- Architecture and service boundaries
- API contracts and integration notes
- Operational workflows
- Release or deployment guidance

## How these repos relate

- `tibs` owns the main public product experience and most transport safety
  workflows.
- `tibs-survey` handles field operations, surveyor workflows, documents,
  projects, expenses, and payout flows.
- `tibs-learning-center` delivers course content, assessments, and learner
  progress.
- All three apps are designed to share the same Supabase project and auth user
  identities.

## Documentation approach

Each repo page should capture:

1. Purpose
2. Setup and environment
3. Current route and feature surface
4. Deployment entrypoints
