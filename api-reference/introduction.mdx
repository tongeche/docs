## What this section covers

This API section is currently organized around the real TIBS domains already
visible in the reference notes:

- Reports
- Incidents
- Vehicles
- Feedback
- Search
- Corridors, routes, and saccos
- Surveyor workflows
- Learning workflows
- Restricted admin flows

## Current source of truth

Until the formal OpenAPI document is ready, use the [API domain map](/api-reference/domain-map)
and the product workflow pages together. They capture intent, likely endpoint
families, and the data surfaces behind them.

The current source repos behind those surfaces are:

- `tibs` for the main transport safety platform and Supabase schema
- `tibs-survey` for surveyor operations and project workflows
- `tibs-learning-center` for course progress and learning flows
