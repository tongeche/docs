## Core pattern

The main pattern visible in the current reference notes is:

- Write raw reports and feedback first
- Process raw records into incidents and summaries
- Read from optimized feeds and summary tables
- Keep restricted admin workflows separate from public-facing reads

## Current implementation ownership

- `tibs` implements the main public transport safety product, including
  reporting, incident views, vehicle lookup, maps, coverage, terms, and
  privacy pages.
- `tibs-survey` implements surveyor operations, assignments, project-scoped
  dashboards, expense claims, payouts, documents, and admin review flows.
- `tibs-learning-center` implements course discovery, lessons, assessments,
  and learning progress.

## Restricted surfaces

Admin and review operations exist, but they should remain clearly marked as
restricted or internal wherever they are documented.
